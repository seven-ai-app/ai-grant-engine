# מדריך אסטרטגי לכתיבת בקשה מנצחת למענק תנופה (7א)

## לקחים ממיזמים מצליחים (כגון קיוז - Queues)

קיוז ניצחו כי הם ידעו לחבר בין בעיה עסקית מוחשית לבין פתרון טכנולוגי עמוק.
הם לא מכרו "פלטפורמה" - הם מכרו פתרון לכאב ספציפי ומדיד.

---

## איך חושב הבוחן (Think Like a Reviewer)

הבוחן שואל 3 שאלות:
1. **האם יש כאן חדשנות אמיתית?** - לא שיפור של קיים, אלא פתרון לבעיה שלא נפתרה עדיין
2. **האם הכאב בשוק כואב מספיק?** - כסף אבוד, זמן שבוזבז, עסקאות שנפלו - נתונים ספציפיים
3. **האם הצוות יכול לבצע?** - ניסיון רלוונטי, הוכחת ביצוע, לא רק תארים

---

## מבנה Elevator Pitch לסיכום מנהלים (עמוד 1)

```
[שם המיזם] פותר את הבעיה של [בעיה ספציפית עם נתון כמותי] שעולה ל[קהל יעד] 
[X שקל / X שעות / X% פגיעה בהכנסות].
הפתרון שלנו הוא [טכנולוגיה ייחודית] שמאפשרת [תוצאה מדידה].
בשוק של [X מיליארד דולר], אנחנו מיצבים את עצמנו כ[מיצוב ייחודי].
המענק יאפשר לנו להשלים [מייסטון טכנולוגי ספציפי] ולהגיע ל[תוצאה עסקית].
```

## כללים לכל סעיף

### כאב השוק - חובה לכמת
- לא: "חברות מתקשות לנהל..."
- כן: "מחקר Gartner 2024 מצא כי 67% מחברות הBtoB מאבדות בממוצע 23% מעסקאות פוטנציאליות בגלל..."
- כן: "שיחה עם 15 CTOs חשפה שצוות DevOps ממוצע מבזבז 12 שעות/שבוע על..."

### טכנולוגיה - חייבת להיות מו"פ, לא הנדסה
- לא: "נפתח אפליקציה שתחבר בין..."
- כן: "האתגר המחקרי המרכזי הוא [בעיה טכנית לא פתורה]. גישתנו לפתרון משלבת [X + Y] בדרך שלא נוסתה קודם."
- דגש: **אי ודאות טכנולוגית** = מחקר. ודאות = הנדסה. הרשות מממנת רק מחקר.

### מיילסטונים - מדידים ולא סובייקטיביים
- לא: "נסיים פיתוח הפלטפורמה"
- כן: "נשיג דיוק של 85% ב-F1-Score על Dataset של 10,000 דגימות"
- כן: "זמן תגובה API < 200ms תחת עומס של 1,000 בקשות/שנייה"
- כן: "3 לקוחות פיילוט עם NPS > 40"

### קניין רוחני - להזכיר תמיד
- סקר פטנטים (Freedom to Operate) - 8,000-15,000 ₪
- פטנט זמני (Provisional Patent) - 5,000-10,000 ₪
- זה מראה רצינות ומגן על ההשקעה

### תרומה לכלכלה הישראלית
- השכר ששולם לעובדים ישראליים
- מיסים שישולמו בישראל
- עסקאות יצוא עתידיות
- מספר משרות שייווצרו

## טבלת מתחרים - איך לעשות אותה נכון

| שם | אתר | מאפיינים | מחיר | נתח שוק | חולשה |
|----|-----|----------|------|---------|--------|
| Competitor A | competitor-a.com | Feature 1, Feature 2 | $X/מוצר | XX% | אין פתרון ל-Y |
| Competitor B | competitor-b.com | Feature 3 | $Y/חודש | XX% | לא מתאים ל-Z |
| הפתרון שלנו | - | כל האמור + X ייחודי | - | - | - |

**חשוב:** לא לכתוב "אין מתחרים" - זה נראה תמים. תמיד יש מתחרים, גם עקיפים (Excel, תהליך ידני, מתחרה בתחום אחר).

## תיקוף שוק (LOI - Letter of Intent)

- אפילו אימייל מ-CTO שאמר "נשמע מעניין, בואו נדבר" = LOI.
- כתוב: "קיבלנו ביטוי עניין מ-3 חברות בתחום ה-[X]: [שם חברה 1] (X עובדים, X מ$ הכנסות), [שם חברה 2]..."
- לא חייב לצרף מסמך רשמי - מספיק לתאר את השיחות

## רשימת בדיקה לפני הגשה (Winning Checklist)

### חדשנות ✓
- [ ] הגדרנו בעיה טכנולוגית שלא נפתרה (State of the Art)
- [ ] הסברנו מה ייחודי בגישה שלנו
- [ ] הדגשנו את אי-הוודאות (זה מה שהופך את זה למו"פ)
- [ ] ציינו TRL התחלתי וסופי

### שוק ✓
- [ ] TAM/SAM/SOM עם מקורות (Gartner/IDC/Statista)
- [ ] כאב כמותי (X% הפסד / X שעות / X$)
- [ ] 3+ ביטויי עניין מלקוחות פוטנציאליים
- [ ] טבלת מתחרים עם URLs ונתחי שוק

### צוות ✓
- [ ] ניסיון רלוונטי ישיר (לא רק תארים)
- [ ] הוכחת ביצוע קודמת
- [ ] פגמים ביכולות + תוכנית לסגירתם (Capability Gaps)

### היתכנות ✓
- [ ] תוכנית עבודה ל-12 חודשים עם 5-7 משימות
- [ ] מיילסטונים מדידים לכל משימה
- [ ] סיכוני מו"פ ופתרונות

### תקציב ✓
- [ ] סך 250,000 ₪ (מקסימום למענק 200,000 ₪)
- [ ] כולל שורת IP (פטנט/סקר)
- [ ] ללא: שכר מייסדים, שכירות, תשתיות כלליות
- [ ] קבלני משנה - תעריף מוצדק (עד 300 ₪/שעה)

### תרומה לישראל ✓
- [ ] ציינו משרות שייווצרו
- [ ] ציינו הכנסות יצוא עתידיות
- [ ] ציינו מסים ותשלומים לרשויות

## ניסוחים שעובדים טוב (Proven Phrases)

### פתיחות חזקות לסעיפים:
- "בשוק ה-[X] הגלובלי, הסוגיה המרכזית שלא נפתרה היא..."
- "מחקר שערכנו עם [N] חברות חשף כי..."
- "האתגר הטכנולוגי המרכזי מסלול פיתוח [X] הוא..."

### ניסוחי חדשנות:
- "טכנולוגית הליבה שלנו מבוססת על [X] - גישה שלא נוסתה קודם בהקשר של [Y]"
- "בניגוד לפתרונות קיימים שמסתמכים על [גישה ישנה], אנו מציעים..."
- "האתגר המחקרי האמיתי הוא [בעיה] - שאלה שהתשובה לה אינה ידועה מראש"

### ניסוחי תקציב:
- "עלות קבלן משנה: [שם/תחום], [N] שעות × 280 ₪/שעה = X ₪, עבור [מה בדיוק]"
- "סקר פטנטים ובדיקת Freedom to Operate: 12,000 ₪ (חיוני לפני כניסה לשוק)"

## לקחי קיוז (Queues) - Case Study

מה שעבד:
1. **בעיה ספציפית**: לא "שיפור תקשורת צוות" אלא "כשל בהעברת הקשר בין משמרות DevOps - עולה X שעות/שבוע ו-Y תקריות/חודש"
2. **טכנולוגיה ייחודית**: עיבוד NLP ספציפי לשפת DevOps - לא GPT כללי אלא מודל fine-tuned על incident reports
3. **מיילסטונים קונקרטיים**: "אלגוריתם יזהה 80% מתקריות חוזרות בתוך 100ms"
4. **תיקוף מוקדם**: 5 LOIs מחברות עם 100+ עובדי DevOps
5. **תקציב מדויק**: 250K עם שורת IP מפורטת

מה שלא עבד בבקשות שנדחו:
- "נפתח פלטפורמה מהפכנית" (ללא הגדרת הבעיה הטכנית)
- מיילסטונים כמו "נסיים פיתוח" (לא מדידים)
- תקציב 150K (לא מנצל את המקסימום - נראה חוסר תכנון)
- אין LOIs - "נאסוף לאחר אישור המענק"
